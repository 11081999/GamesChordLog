Adaptive pattern


  • Justificación:
  • Para la construcción del servicio web que proporcionará GamesChord se va a utilizar dos APIs: Steam API y Spotify API desde las cuales mayormente se solicitara información con la que trabaja el algoritmo principal de recomendación de música. Dicha información no viene organizada para satisfacer específicamente las necesidades de este proyecto, por lo que se tendría que necesitar una API propia u una librería que pueda organizar los datos de tal forma que el algoritmo lo pueda aceptar y procesar.

Lyered Pattern


  • Justificación:
  • Para el control en general de todo le d¿flujo de información que pasa a través del aplicativo software y en general de la interacción con el usuario se debe usar una logia ce capas, ya que toda la información debe de ser procesada antes de guardarla en la base de datos y antes de presentársela al usuario. Esto por temas de seguridad, autenticación y aplicación del algoritmo que recomienda la música el cual es el servicio no esencial de mayor prioridad.

Composite Pattern


  • Justificación:
  • El proyecto de GamesChord está desarrollado en node JS, por lo que se utilizará la estructura estándar que se utiliza a nivel mundial. Esta consiste en seccionar las diferentes funcionalidades del proyecto en folders designados (Controllers, Routes, Views, etc). El patrón compuesto nos sirve para definir los diferentes elementos que se pueden o se van a integrar a cada vista en particular. Por ejemplo, el header de la página web generalmente se quiere que sea persistente pero no se va a copiar y pegar el mismo código del header en cada viewer sino que se hace un bloque que se llama en cada view en donde se quiera tener el header.

Proxy Pattern


  • Justificación:
  • Se puede considerar un patrón de Proxy (proxy remoto) al momento de tener un carpeta de controllers ya que el usuario no interactúa directamente con la API sino que primero su la route request handler pasa primero por los controllers como node JS dictamina. Y a parte se tiene que hacer un llamado desde las API de Steam o de Spotify hacia sus servidores para obtener la información de las cuentas del usuario en esos servicios ajenos al proyecto. De igual forma se puede considerar un Proxy virtual que guarde una persistencia de toda la información de la cuenta del usuario en Steam Y Spotify para poder así consultarla sin tener que hacer una llamada cada.
